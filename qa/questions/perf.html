pprof 是 Go 内置的性能分析工具，可以分析 CPU、内存、goroutine 等。

// CPU分析
go tool pprof http://localhost:6060/debug/pprof/profile
// 内存分析
go tool pprof http://localhost:6060/debug/pprof/heap
// goroutine分析
go tool pprof http://localhost:6060/debug/pprof/goroutine

实际案例：

  • 服务卡顿 → pprof CPU 分析发现签名校验模块耗时过长，优化签名算法
  • 内存占用高 → pprof heap 分析发现大量 goroutine，改成 worker pool 模式

补充：pprof 进阶工具：

  • go tool trace 可视化分析 goroutine 调度、GC、系统调用等事件
  • 火焰图：横轴表示采样宽度（越宽越耗时），纵轴表示调用栈深度
  • block profile：分析 goroutine 等待锁/channel 的时间
  • mutex profile：分析锁竞争热点

问题：抢购活动被脚本攻击，有人直接用脚本刷接口抢购

方案：客户端增加参数签名验证

  • 客户端用密钥对请求参数生成签名
  • 服务端校验签名合法性，不合法直接拒绝
  • 直接刷掉了所有通过脚本进行的抢购攻击

后续问题：服务变得很卡顿

  • 通过 pprof 分析发现是签名校验模块耗时过长
  • 优化了签名生成机制（简化算法、缓存密钥等）
  • 用 benchmark 测试调优，确保校验性能

补充：多层防护策略：

  • IP 限流：单 IP 请求频率超过阈值自动封禁
  • 行为分析：检测异常请求模式（如固定间隔、无用户行为轨迹）
  • 验证码：关键操作触发验证码，拦截自动化脚本
  • WAF：Web 应用防火墙，拦截常见攻击模式

💼 实际案例

项目中大量异步任务用 channel + go 处理，每个任务创建一个 goroutine，通过 pprof 发现内存占用过高，改成 worker pool 后内存大幅下降。

补充：常见 goroutine 泄漏场景：

  • channel 发送/接收无人接收，导致 goroutine 永久阻塞
  • context 未取消，goroutine 等待资源释放
  • timer 未 Stop，导致 goroutine 堆积
  • sync.WaitGroup 计数不匹配，Wait 永远不返回

固定数量的 worker goroutine，每个 worker 从任务 channel 中取任务执行。

func WorkerPool(workers int, tasks <-chan Task) {
    var wg sync.WaitGroup
    for i := 0; i < workers; i++ {
        wg.Add(1)
        go func() {
            defer wg.Done()
            for task := range tasks {
                task.Process()
            }
        }()
    }
    wg.Wait()
}
  • 优点：控制 goroutine 数量，减少内存占用，避免频繁创建销毁
  • 对比：直接 go func() 每个任务一个 goroutine，简单但不可控

补充：进阶能力：

  • 动态扩缩：根据任务队列长度动态调整 worker 数量
  • 优雅关闭：通过 context 取消 + channel 关闭，等待所有 worker 完成当前任务后退出
  • 任务超时：每个任务设置独立超时，避免单个任务卡住整个 worker
  • 错误处理：任务失败重试策略，死信队列存储多次失败的任务
